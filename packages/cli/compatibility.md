# YAML Compatibility

Agent CI aims to run real GitHub Actions workflows locally. The table below shows current support against the [official workflow syntax](https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions).

‚úÖ = Supported &nbsp; ‚ö†Ô∏è = Partial &nbsp; ‚ùå = Not supported &nbsp; üü° = Ignored (no-op)

## Workflow-Level Keys

| Key                                | Status | Notes                                                                                                                                                                                                 |
| ---------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`                             | ‚úÖ     |                                                                                                                                                                                                       |
| `run-name`                         | üü°     | Parsed but not displayed anywhere                                                                                                                                                                     |
| `on` (push, pull_request)          | ‚úÖ     | Branch and path filters are evaluated when using `--all`                                                                                                                                              |
| `on` (schedule, workflow_dispatch) | üü°     | Accepted without error, but Agent CI does not simulate event triggers ‚Äî workflows must be run manually                                                                                                |
| `on` (workflow_call)               | ‚ö†Ô∏è     | Local reusable workflows (`uses: ./.github/workflows/...`) are inlined into the caller's dependency graph. Remote refs, `inputs:`/`outputs:` passing, and nested reusable workflows are not supported |
| `on` (other events)                | üü°     | Parsed without error, but the event is not simulated                                                                                                                                                  |
| `env`                              | ‚úÖ     | Workflow-level env is propagated to all steps                                                                                                                                                         |
| `defaults.run.shell`               | ‚úÖ     | Passed through to the runner                                                                                                                                                                          |
| `defaults.run.working-directory`   | ‚úÖ     | Passed through to the runner                                                                                                                                                                          |
| `permissions`                      | üü°     | Accepted but not enforced ‚Äî the mock GITHUB_TOKEN has full access                                                                                                                                     |
| `concurrency`                      | ‚ùå     | Concurrency groups are a GitHub-side queuing and cancellation mechanism. Agent CI has no persistent server to track group state across runs, so this cannot be implemented locally                    |

## Job-Level Keys

| Key                                   | Status | Notes                                                                                                                                                                                                                        |
| ------------------------------------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `jobs.<id>`                           | ‚úÖ     | Multiple jobs in a single workflow                                                                                                                                                                                           |
| `jobs.<id>.name`                      | ‚úÖ     |                                                                                                                                                                                                                              |
| `jobs.<id>.needs`                     | ‚úÖ     | Jobs are sorted topologically into dependency waves                                                                                                                                                                          |
| `jobs.<id>.if`                        | ‚ö†Ô∏è     | Supported: `success()`, `failure()`, `always()`, `cancelled()`, `==`/`!=`, `&&`/`\|\|`, `needs.*.outputs.*`, `needs.*.result`. Not supported: `contains()`, `startsWith()`, `endsWith()`, unary `!`, and numeric comparisons |
| `jobs.<id>.runs-on`                   | üü°     | Accepted but always runs in a Linux container regardless of the value                                                                                                                                                        |
| `jobs.<id>.environment`               | üü°     | Accepted but not enforced ‚Äî environment protection rules are GitHub-side only                                                                                                                                                |
| `jobs.<id>.env`                       | ‚úÖ     |                                                                                                                                                                                                                              |
| `jobs.<id>.defaults.run`              | ‚úÖ     | `shell` and `working-directory`                                                                                                                                                                                              |
| `jobs.<id>.outputs`                   | ‚úÖ     | Resolved after each job completes and accumulated across dependency waves                                                                                                                                                    |
| `jobs.<id>.timeout-minutes`           | ‚ùå     | Not implemented. Agent CI's pause-on-failure model is the intended way to handle long-running steps ‚Äî a hard timeout would destroy the container state that makes local debugging possible                                   |
| `jobs.<id>.continue-on-error`         | ‚ùå     | Not implemented. Agent CI pauses on failure so you can inspect and fix the container in place; `continue-on-error` would skip past failures and discard that debugging opportunity                                           |
| `jobs.<id>.concurrency`               | ‚ùå     | See workflow-level `concurrency` above                                                                                                                                                                                       |
| `jobs.<id>.container`                 | ‚úÖ     | Short and long form; image, env, ports, volumes, and options are all supported                                                                                                                                               |
| `jobs.<id>.services`                  | ‚úÖ     | Sidecar containers with image, env, ports, and options                                                                                                                                                                       |
| `jobs.<id>.uses` (reusable workflows) | ‚ö†Ô∏è     | Local refs (`./`) are expanded inline. Remote refs are skipped with a warning. `with:` (inputs) and `secrets:` pass-through are not yet supported                                                                            |
| `jobs.<id>.secrets`                   | ‚ùå     | Agent CI cannot access GitHub's secret storage. Use a `.env.agent-ci` file at the project root instead ‚Äî secrets are loaded from there and injected as `${{ secrets.* }}` expressions                                        |

## Strategy / Matrix

| Key                       | Status | Notes                                                                                                                                                                                               |
| ------------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `strategy.matrix`         | ‚úÖ     | Cartesian product of all array values is fully expanded                                                                                                                                             |
| `strategy.matrix.include` | ‚ùå     | Not implemented. The matrix parser only processes array-valued keys; `include` entries (which are objects) are silently dropped. Adding support would require post-processing the Cartesian product |
| `strategy.matrix.exclude` | ‚ùå     | Not implemented ‚Äî same reason as `include`. `exclude` entries are objects and are dropped by the array-only parser                                                                                  |
| `strategy.fail-fast`      | ‚úÖ     | Setting `fail-fast: false` allows remaining matrix jobs to continue after a failure                                                                                                                 |
| `strategy.max-parallel`   | ‚ùå     | Not implemented. Parallelism is controlled by Agent CI's host-level concurrency limiter (based on CPU count), not per-workflow job limits                                                           |

| Key                          | Status | Notes                                                                                                                                                                                                                                                       |
| ---------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `steps[*].id`                | ‚úÖ     |                                                                                                                                                                                                                                                             |
| `steps[*].name`              | ‚úÖ     | Expression expansion in names                                                                                                                                                                                                                               |
| `steps[*].if`                | ‚ö†Ô∏è     | The condition is passed to the official runner binary, which evaluates it at runtime. Limitation: `steps.*.outputs.cache-hit` and similar outputs resolve to an empty string at parse time because prior steps have not yet run when the workflow is parsed |

| `steps[*].uses`              | ‚úÖ     | Public actions are downloaded via the GitHub API                                                                                                                                                                                                            |
| `steps[*].uses` (local `./`) | ‚úÖ     | Local actions (e.g. `uses: ./.github/actions/my-action`) are resolved from the workspace                                                                                                                                                                    |
| `steps[*].with`              | ‚úÖ     | Expression expansion in values                                                                                                                                                                                                                              |
| `steps[*].env`               | ‚úÖ     | Expression expansion in values                                                                                                                                                                                                                              |
| `steps[*].working-directory` | ‚úÖ     |                                                                                                                                                                                                                                                             |
| `steps[*].shell`             | ‚úÖ     | Passed through to the runner                                                                                                                                                                                                                                |
| `steps[*].continue-on-error` | ‚ùå     | Not implemented ‚Äî see `jobs.<id>.continue-on-error` above for the reasoning                                                                                                                                                                                 |
| `steps[*].timeout-minutes`   | ‚ùå     | Not implemented ‚Äî see `jobs.<id>.timeout-minutes` above for the reasoning                                                                                                                                                                                   |

## Expressions (`${{ }}`)

| Expression                                          | Status | Notes                                                                                                                                                                         |
| --------------------------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `hashFiles(...)`                                    | ‚úÖ     | SHA-256 of matching files; supports multiple glob patterns                                                                                                                    |
| `format(...)`                                       | ‚úÖ     | Template substitution with recursive expression expansion                                                                                                                     |
| `matrix.*`                                          | ‚úÖ     |                                                                                                                                                                               |
| `secrets.*`                                         | ‚úÖ     | Loaded from `.env.agent-ci` at the project root                                                                                                                               |
| `runner.os`                                         | ‚úÖ     | Always returns `Linux`                                                                                                                                                        |
| `runner.arch`                                       | ‚úÖ     | Always returns `X64`                                                                                                                                                          |
| `github.sha`, `github.ref_name`, etc.               | ‚ö†Ô∏è     | Returns hardcoded dummy values: `sha` is all zeros, `ref_name` and `head_ref` are `main`, `repository` is `local/repo`, `actor` is `local`, `run_id` and `run_number` are `1` |
| `github.event.*`                                    | ‚ö†Ô∏è     | All event payload fields return empty strings ‚Äî no real webhook event is triggered locally                                                                                    |
| `strategy.job-total`, `strategy.job-index`          | ‚úÖ     |                                                                                                                                                                               |
| `steps.*.outputs.*`                                 | ‚ö†Ô∏è     | Resolves to an empty string at parse time. The official runner evaluates these correctly at runtime ‚Äî the limitation only affects Agent CI's own expression pre-processing    |
| `needs.*.outputs.*`                                 | ‚úÖ     | Resolved after dependency jobs complete. The needs context is built from actual job outputs and passed into subsequent job evaluation                                         |
| Boolean/comparison operators                        | ‚ö†Ô∏è     | Supported in job-level `if`: `==`, `!=`, `&&`, `\|\|`, parentheses. Not supported: unary `!`, numeric comparisons (`<`, `>`, `<=`, `>=`)                                      |
| `toJSON`, `fromJSON`                                | ‚úÖ     |                                                                                                                                                                               |
| `contains`, `startsWith`, `endsWith`                | ‚ùå     | Not implemented in the expression parser. The evaluator handles context lookups and comparison operators but does not support arbitrary function calls with string arguments  |
| `success()`, `failure()`, `always()`, `cancelled()` | ‚úÖ     | Evaluated by Agent CI for job-level `if` conditions                                                                                                                           |

## GitHub API Features (DTU Mock)

| Feature                                                 | Status | Notes                                                                                                                 |
| ------------------------------------------------------- | ------ | --------------------------------------------------------------------------------------------------------------------- |
| Action downloads                                        | ‚úÖ     | Action tarballs are resolved and downloaded from github.com                                                           |
| `actions/cache`                                         | ‚úÖ     | Cache is stored on the local filesystem via bind-mount, giving ~0 ms round-trip on cache hits                         |
| `actions/checkout`                                      | ‚úÖ     | The workspace is rsynced into the container with `clean: false` to preserve local changes                             |
| `actions/setup-node`, `actions/setup-python`, etc.      | ‚úÖ     | Tool setup actions run natively inside the runner container                                                           |
| `actions/upload-artifact` / `download-artifact`         | ‚úÖ     | Artifacts are stored on the local filesystem                                                                          |
| `GITHUB_TOKEN`                                          | ‚úÖ     | A mock token is injected; all GitHub API calls from the runner are answered locally by Agent CI's API emulation layer |
| Workflow commands (`::set-output::`, `::error::`, etc.) | ‚úÖ     | Handled by the official runner binary                                                                                 |
