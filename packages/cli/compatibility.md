# YAML Compatibility

Agent CI aims to run real GitHub Actions workflows locally. The table below shows current support against the [official workflow syntax](https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions).

тЬЕ = Supported &nbsp; тЪая╕П = Partial &nbsp; тЭМ = Not supported &nbsp; ЁЯЯб = Ignored (no-op) &nbsp; ЁЯЪл = Not planned

## Workflow-Level Keys

| Key                                | Status | Notes                                                                                                                                                                                                                                                                           |
| ---------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`                             | тЬЕ     |                                                                                                                                                                                                                                                                                 |
| `run-name`                         | ЁЯЯб     | Parsed but not displayed anywhere                                                                                                                                                                                                                                               |
| `on` (push, pull_request)          | тЬЕ     | Branch and path filters are evaluated when using `--all`                                                                                                                                                                                                                        |
| `on` (schedule, workflow_dispatch) | ЁЯЯб     | Accepted without error, but Agent CI does not simulate event triggers тАФ workflows must be run manually                                                                                                                                                                          |
| `on` (workflow_call)               | тЬЕ     | Local reusable workflows (`uses: ./.github/workflows/...`) are inlined into the caller's dependency graph. Remote refs are fetched from GitHub (requires `--github-token` or `AGENT_CI_GITHUB_TOKEN`). `inputs:`/`outputs:` passing and nested reusable workflows are supported |
| `on` (other events)                | ЁЯЯб     | Parsed without error, but the event is not simulated                                                                                                                                                                                                                            |
| `env`                              | тЬЕ     | Workflow-level env is propagated to all steps                                                                                                                                                                                                                                   |
| `defaults.run.shell`               | тЬЕ     | Non-bash shells (`sh`, `python`, `pwsh`) are invoked via a parse-time heredoc wrap because the runner ignores `inputs.shell` for script steps. `bash` is the default.                                                                                                           |
| `defaults.run.working-directory`   | тЬЕ     | Passed through to the runner                                                                                                                                                                                                                                                    |
| `permissions`                      | ЁЯЯб     | Accepted but not enforced тАФ the mock GITHUB_TOKEN has full access                                                                                                                                                                                                               |
| `concurrency`                      | ЁЯЪл     | Concurrency groups are a GitHub-side queuing and cancellation mechanism. Agent CI has no persistent server to track group state across runs, so this cannot be implemented locally                                                                                              |

## Job-Level Keys

| Key                                   | Status | Notes                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `jobs.<id>`                           | тЬЕ     | Multiple jobs in a single workflow                                                                                                                                                                                                                                                                                                                                                                                            |
| `jobs.<id>.name`                      | тЬЕ     |                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `jobs.<id>.needs`                     | тЬЕ     | Jobs are sorted topologically into dependency waves                                                                                                                                                                                                                                                                                                                                                                           |
| `jobs.<id>.if`                        | тЬЕ     | `success()`, `failure()`, `always()`, `cancelled()`, `==`/`!=`, `&&`/`\|\|`, `!`, `needs.*.outputs.*`, `needs.*.result`                                                                                                                                                                                                                                                                                                       |
| `jobs.<id>.runs-on`                   | тЪая╕П     | `ubuntu-*` / `linux` (and unspecified `self-hosted` / custom labels) run in a Linux container тАФ the default image is minimal, see [runner-image.md](./runner-image.md) to add system tools. `macos-*` runs in a real macOS VM on Apple Silicon hosts with [tart](https://github.com/cirruslabs/tart) + `sshpass` installed; other hosts skip the job with a reason. `windows-*` is not yet supported and skips on every host. |
| `jobs.<id>.environment`               | ЁЯЯб     | Accepted but not enforced тАФ environment protection rules are GitHub-side only                                                                                                                                                                                                                                                                                                                                                 |
| `jobs.<id>.env`                       | тЬЕ     |                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `jobs.<id>.defaults.run`              | тЬЕ     | `shell` and `working-directory`                                                                                                                                                                                                                                                                                                                                                                                               |
| `jobs.<id>.outputs`                   | тЬЕ     | Resolved after each job completes and accumulated across dependency waves                                                                                                                                                                                                                                                                                                                                                     |
| `jobs.<id>.timeout-minutes`           | тЭМ     | Not implemented. Agent CI's pause-on-failure model is the intended way to handle long-running steps тАФ a hard timeout would destroy the container state that makes local debugging possible                                                                                                                                                                                                                                    |
| `jobs.<id>.continue-on-error`         | тЭМ     | Not implemented. Agent CI pauses on failure so you can inspect and fix the container in place; `continue-on-error` would skip past failures and discard that debugging opportunity                                                                                                                                                                                                                                            |
| `jobs.<id>.concurrency`               | ЁЯЪл     | See workflow-level `concurrency` above                                                                                                                                                                                                                                                                                                                                                                                        |
| `jobs.<id>.container`                 | тЪая╕П     | Short and long form; `image`, `env`, `ports`, `volumes` honored. Within `options:`, only `--env`/`-e` and `--label`/`-l` are forwarded to the runner container; other Docker flags (`--privileged`, `--user`, `--network`, `--cap-add`, `--workdir`, etc.) are silently ignored because they tend to clash with agent-ci's own container orchestration.                                                                       |
| `jobs.<id>.services`                  | тЬЕ     | Sidecar containers with image, env, ports, and options                                                                                                                                                                                                                                                                                                                                                                        |
| `jobs.<id>.uses` (reusable workflows) | тЬЕ     | Local refs (`./`) are expanded inline. Remote refs are fetched from GitHub (requires `--github-token` or `AGENT_CI_GITHUB_TOKEN`; public repos may work without auth). `with:` (inputs) and `secrets:` pass-through are supported                                                                                                                                                                                             |
| `jobs.<id>.secrets`                   | ЁЯЪл     | Agent CI cannot access GitHub's secret storage. Secrets are resolved from a `.env.agent-ci` file at the project root or from shell environment variables as fallback. `--github-token` / `AGENT_CI_GITHUB_TOKEN` auto-populates `secrets.GITHUB_TOKEN`. All are injected as `${{ secrets.* }}` expressions                                                                                                                    |

## Strategy / Matrix

| Key                       | Status | Notes                                                                                                                                                                                               |
| ------------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `strategy.matrix`         | тЬЕ     | Cartesian product of all array values is fully expanded                                                                                                                                             |
| `strategy.matrix.include` | тЭМ     | Not implemented. The matrix parser only processes array-valued keys; `include` entries (which are objects) are silently dropped. Adding support would require post-processing the Cartesian product |
| `strategy.matrix.exclude` | тЭМ     | Not implemented тАФ same reason as `include`. `exclude` entries are objects and are dropped by the array-only parser                                                                                  |
| `strategy.fail-fast`      | тЬЕ     | Setting `fail-fast: false` allows remaining matrix jobs to continue after a failure                                                                                                                 |
| `strategy.max-parallel`   | тЭМ     | Not implemented. Parallelism is controlled by Agent CI's host-level concurrency limiter (based on CPU count), not per-workflow job limits                                                           |

## Step-Level Keys

| Key                          | Status | Notes                                                                                                                                                                                                                                                       |
| ---------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `steps[*].id`                | тЬЕ     |                                                                                                                                                                                                                                                             |
| `steps[*].name`              | тЬЕ     | Expression expansion in names                                                                                                                                                                                                                               |
| `steps[*].if`                | тЪая╕П     | The condition is passed to the official runner binary, which evaluates it at runtime. Limitation: `steps.*.outputs.cache-hit` and similar outputs resolve to an empty string at parse time because prior steps have not yet run when the workflow is parsed |
| `steps[*].run`               | тЬЕ     | Multiline shell scripts with `${{ }}` expression expansion                                                                                                                                                                                                  |
| `steps[*].uses`              | тЬЕ     | Public actions are downloaded via the GitHub API                                                                                                                                                                                                            |
| `steps[*].uses` (local `./`) | тЬЕ     | Local actions (e.g. `uses: ./.github/actions/my-action`) are resolved from the workspace                                                                                                                                                                    |
| `steps[*].with`              | тЬЕ     | Expression expansion in values                                                                                                                                                                                                                              |
| `steps[*].env`               | тЬЕ     | Expression expansion in values                                                                                                                                                                                                                              |
| `steps[*].working-directory` | тЬЕ     |                                                                                                                                                                                                                                                             |
| `steps[*].shell`             | тЬЕ     | See `defaults.run.shell` above тАФ the same heredoc-wrap mechanism applies.                                                                                                                                                                                   |
| `steps[*].continue-on-error` | тЭМ     | Not implemented тАФ see `jobs.<id>.continue-on-error` above for the reasoning                                                                                                                                                                                 |
| `steps[*].timeout-minutes`   | тЭМ     | Not implemented тАФ see `jobs.<id>.timeout-minutes` above for the reasoning                                                                                                                                                                                   |

## Expressions (`${{ }}`)

| Key                                                 | Status | Notes                                                                                                                                                                                                                                                                                                                 |
| --------------------------------------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `hashFiles(...)`                                    | тЬЕ     | SHA-256 of matching files; supports multiple glob patterns. Descends into dotted directories (e.g. `.github/`) when the pattern asks for them.                                                                                                                                                                        |
| `format(...)`                                       | тЬЕ     | Template substitution with recursive expression expansion                                                                                                                                                                                                                                                             |
| `matrix.*`                                          | тЬЕ     |                                                                                                                                                                                                                                                                                                                       |
| `secrets.*`                                         | тЬЕ     | Resolved from `.env.agent-ci` at the project root, with shell environment variables as fallback. `--github-token` auto-populates `secrets.GITHUB_TOKEN`                                                                                                                                                               |
| `runner.os`                                         | тЬЕ     | `Linux` for container-based jobs, `macOS` for jobs routed to the macOS VM                                                                                                                                                                                                                                             |
| `runner.arch`                                       | тЬЕ     | `X64` for container-based jobs, `ARM64` for macOS VM jobs (Apple Silicon)                                                                                                                                                                                                                                             |
| `github.sha`, `github.ref_name`, etc.               | тЪая╕П     | `sha` is the real commit SHA from `HEAD`. When the working tree is dirty, a temporary tree SHA is computed to reflect uncommitted changes. Other values use defaults: `ref_name` and `head_ref` are `main`, `repository` is derived from the git remote, `actor` is the repo owner, `run_id` and `run_number` are `1` |
| `github.event.*`                                    | тЪая╕П     | All event payload fields return empty strings тАФ no real webhook event is triggered locally                                                                                                                                                                                                                            |
| `strategy.job-total`, `strategy.job-index`          | тЬЕ     |                                                                                                                                                                                                                                                                                                                       |
| `steps.*.outputs.*`                                 | тЪая╕П     | Always resolves to an empty string at parse time тАФ the producing step has not run yet and the runner does not re-evaluate `${{ }}` inside `run:` script bodies. Use `needs.*.outputs.*` for cross-job values instead.                                                                                                 |
| `needs.*.outputs.*`                                 | тЬЕ     | Resolved after dependency jobs complete. The needs context is built from actual job outputs and passed into subsequent job evaluation                                                                                                                                                                                 |
| Boolean/comparison operators                        | тЬЕ     | `==`, `!=`, `<`, `>`, `<=`, `>=`, `&&`, `\|\|`, `!`, parentheses. Case-insensitive string comparison. Numeric coercion follows GitHub Actions semantics: empty string and `null` become `0`, numeric-looking strings are parsed, anything else becomes `NaN` and compares false.                                      |
| `toJSON`, `fromJSON`                                | тЬЕ     | `toJSON` emits 2-space-indented JSON matching GitHub Actions' pretty-printed output, so `hashFiles`/diff uses of `toJSON(x)` are reproducible.                                                                                                                                                                        |
| `contains`, `startsWith`, `endsWith`                | тЬЕ     | Case-insensitive. `contains` works with both strings and JSON arrays                                                                                                                                                                                                                                                  |
| `join`                                              | тЬЕ     | Joins JSON arrays with a separator (default `, `)                                                                                                                                                                                                                                                                     |
| `success()`, `failure()`, `always()`, `cancelled()` | тЬЕ     | Evaluated by Agent CI for job-level `if` conditions and in expression expansion                                                                                                                                                                                                                                       |
| Type literals (`true`, `false`, `null`, numbers)    | тЬЕ     | Boolean, null, and numeric literals supported in expressions                                                                                                                                                                                                                                                          |

## GitHub API Features (DTU Mock)

| Key                                                     | Status | Notes                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Action downloads                                        | тЬЕ     | Action tarballs are resolved and downloaded from github.com                                                                                                                                                                                                                                                                                                                                     |
| `actions/cache`                                         | тЪая╕П     | Cache is stored on the local filesystem via bind-mount, giving ~0 ms round-trip on cache hits. Scoping is by cache key only тАФ GitHub's ref-based isolation (a branch cannot read another branch's cache, default branch is a fallback) is not implemented. If two workflows on different refs share a key they share the cache.                                                                 |
| `actions/checkout`                                      | тЬЕ     | The workspace is rsynced into the container with `clean: false` to preserve local changes                                                                                                                                                                                                                                                                                                       |
| `actions/setup-node`, `actions/setup-python`, etc.      | тЬЕ     | Tool setup actions run natively inside the runner container                                                                                                                                                                                                                                                                                                                                     |
| `actions/upload-artifact` / `download-artifact`         | тЬЕ     | Artifacts are stored on the local filesystem                                                                                                                                                                                                                                                                                                                                                    |
| `GITHUB_TOKEN`                                          | тЪая╕П     | Not injected automatically. Pass `--github-token` or set `AGENT_CI_GITHUB_TOKEN` to make it available as `secrets.GITHUB_TOKEN`. GitHub API calls from the runner are answered locally by Agent CI's API emulation layer. OIDC ID-token issuance (`permissions.id-token: write` тЖТ federated cloud auth) is not implemented тАФ actions that exchange the OIDC token with AWS/GCP/Vault will fail. |
| Workflow commands (`::set-output::`, `::error::`, etc.) | тЬЕ     | Handled by the official runner binary                                                                                                                                                                                                                                                                                                                                                           |
